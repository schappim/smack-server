API Reference

Complete reference for all Smack REST API endpoints

Overview

Base URL

http://localhost:8080

All API requests must include a valid JWT token in the Authorization header, except for public endpoints like register, login, and health check.

Authorization Header
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Error Handling

All errors return a JSON object with an error field:

Error Response
{
  "error": "Error message description"
}
Status Code Description
400Bad Request - Invalid input
401Unauthorized - Missing or invalid token
403Forbidden - Not allowed to perform action
404Not Found - Resource doesn't exist
409Conflict - Duplicate resource
500Internal Server Error

Channels

GET /api/channels
List channels the current user is a member of
Response
[
  {
    "id": "uuid",
    "name": "general",
    "description": "General discussion",
    "is_direct": false,
    "created_by": "uuid",
    "created_at": "2024-01-15T10:30:00Z",
    "unread_count": 5
  }
]
GET /api/channels/public
List all public (non-direct) channels
POST /api/channels
Create a new channel
ParameterTypeDescription
name required string Channel name
description optional string Channel description
Request
curl -X POST http://localhost:8080/api/channels \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "engineering", "description": "Engineering team discussions"}'
GET /api/channels/{id}
Get channel details
POST /api/channels/{id}/join
Join a channel
POST /api/channels/{id}/leave
Leave a channel
GET /api/channels/{id}/members
Get all members of a channel

Channel Actions

/api/channels/{id}/unmutePOSTUnmute channel
/api/channels/{id}/clearPOSTClear all messages
/api/channels/mutedGETList muted channel IDs
POST /api/dm
Create or get existing DM channel with another user
ParameterTypeDescription
user_id required string UUID of the user to message

Reminders

GET /api/reminders
List all pending reminders for the current user
POST /api/reminders
Create a new reminder
ParameterTypeDescription
channel_id requiredstringChannel to post reminder in
message requiredstringReminder message
remind_atstringISO8601 timestamp
remind_instringRelative time (e.g., "30m", "2h", "1d")
Example with relative time
curl -X POST http://localhost:8080/api/reminders \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "channel_id": "CHANNEL_UUID",
    "message": "Stand-up meeting!",
    "remind_in": "30m"
  }'
DELETE /api/reminders/{id}
Delete a reminder

Files

POST /api/files/upload
Upload a file (max 10MB). Supported: JPEG, PNG, GIF, WebP, SVG
Request
curl -X POST http://localhost:8080/api/files/upload \
  -H "Authorization: Bearer TOKEN" \
  -F "file=@/path/to/image.png"
Response
{
  "url": "/api/files/abc123-image.png",
  "filename": "abc123-image.png"
}
GET /api/files/{filename}
Retrieve an uploaded file

GET /api/bots/{id}
Get bot details
POST /api/bots/dm
Create a DM channel with a bot
ParameterTypeDescription
bot_id required string Bot UUID

After creating a DM with a bot, you can send messages to it like any other channel. The bot will respond with AI-generated content, streamed via WebSocket.
