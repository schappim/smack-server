Webhooks

Post messages from external services with rich HTML widgets

Overview

Webhooks allow external services to post messages to Smack channels. Each webhook has a unique URL with a secret token that acts as authentication.

Use Cases

GitHub notifications, CI/CD deploy status, monitoring alerts, scheduled reports, external bot integrations, and any service that can make HTTP POST requests.

Managing Webhooks

Create a Webhook

POST /api/webhooks
Create a new incoming webhook for a channel. Requires authentication.
ParameterTypeDescription
name required string Display name for the webhook
channel_id required string Target channel UUID
Request
curl -X POST http://localhost:8080/api/webhooks \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "GitHub Notifications",
    "channel_id": "CHANNEL_UUID"
  }'
Response201 Created
{
  "id": "webhook-uuid",
  "name": "GitHub Notifications",
  "channel_id": "channel-uuid",
  "token": "secret-token-string",
  "created_by": "user-uuid",
  "created_at": "2024-01-15T10:30:00Z",
  "url": "/api/webhooks/incoming/webhook-uuid/secret-token-string"
}

Keep Your Token Secret!

The webhook URL contains a secret token. Anyone with this URL can post messages to your channel. Treat it like a password.

List Webhooks

GET /api/webhooks
List all webhooks. Optionally filter by channel.
Query ParamDescription
channel_idFilter by channel UUID (optional)

Get / Delete Webhook

GET /api/webhooks/{id}
Get webhook details
DELETE /api/webhooks/{id}
Delete a webhook

Posting Messages via Webhook

Use the incoming webhook URL to post messages. No authentication header required—the token in the URL acts as authentication.

POST /api/webhooks/incoming/{id}/{token}
Post a message via webhook. No auth header required.
ParameterTypeDescription
content required string Plain text message content
html optional string HTML content for rich widget display
widget_size optional string Widget size: small, medium, large, xlarge
username optional string Override the display name
avatar_url optional string Override the avatar image

Simple Text Message

Example: Plain text
curl -X POST http://localhost:8080/api/webhooks/incoming/WEBHOOK_ID/TOKEN \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Build #1234 completed successfully!",
    "username": "CI Bot"
  }'

Interactive Widgets

Widgets can include JavaScript for interactivity. The code runs in a sandboxed iframe.

Interactive Counter Widget
{
  "content": "Interactive Counter",
  "widget_size": "large",
  "html": "<div id='counter-widget'>...</div><script>let count = 0; function increment() { count++; update(); }...</script>"
}

Widget Best Practices

1. Include all styles inline for portability
2. Use system fonts (-apple-system, sans-serif) for consistency
3. Add border-radius: 12px for the Smack aesthetic
4. Test on both light and dark backgrounds
5. Keep widgets under 50KB for fast loading

Integration Examples

GitHub Actions

.github/workflows/notify.yml
name: Notify Smack
on: [push]
jobs:
  notify:
    runs-on: ubuntu-latest
    steps:
      - name: Send notification
        run: |
          curl -X POST ${{ secrets.SMACK_WEBHOOK_URL }} \
            -H "Content-Type: application/json" \
            -d '{"content": "Push to ${{ github.ref }}", "username": "GitHub"}'

Node.js

JavaScript Example
const webhookUrl = 'http://localhost:8080/api/webhooks/incoming/ID/TOKEN';

async function sendAlert(message) {
  await fetch(webhookUrl, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      content: message,
      username: 'Alert Bot',
      widget_size: 'small',
      html: `<div style="background: #ef4444; color: white; padding: 12px; border-radius: 8px;">${message}</div>`
    })
  });
}

sendAlert('Server CPU at 95%!');
