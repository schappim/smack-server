Kanban Boards

Full project management with boards, columns, cards, labels, and comments

Data Model

Kanban boards follow a hierarchical structure:

Board

Project container

→

Column

Status/stage

→

Card

Task/item

Cards can have labels, assignees, due dates, and comments. Each board has its own set of labels and members.

Boards

GET /api/boards
List all boards you're a member of
POST /api/boards
Create a new board
ParameterTypeDescription
name requiredstringBoard name
description optionalstringBoard description
Board Response
{
  "id": "board-uuid",
  "name": "Project Alpha",
  "description": "Main development board",
  "created_by": "user-uuid",
  "created_at": "2024-01-15T10:30:00Z",
  "columns": [
    {
      "id": "col-uuid",
      "name": "To Do",
      "position": 0,
      "cards": [...]
    }
  ]
}
GET /api/boards/{id}
Get board with all columns and cards
PUT /api/boards/{id}
Update board name/description
DELETE /api/boards/{id}
Delete board and all its contents

Board Members

MethodEndpointDescription
GET/api/boards/{id}/membersList board members
POST/api/boards/{id}/membersAdd member (body: {"user_id": "uuid"})
DELETE/api/boards/{id}/members/{userId}Remove member

Columns

Columns represent stages in your workflow (e.g., To Do, In Progress, Done).

POST /api/boards/{id}/columns
Create a new column
ParameterTypeDescription
name requiredstringColumn name
PUT /api/columns/{id}
Update column name
DELETE /api/columns/{id}
Delete column and all its cards
POST /api/boards/{id}/columns/reorder
Reorder columns
ParameterTypeDescription
column_ids requiredarrayOrdered array of column UUIDs

Cards

Cards are the individual tasks or items on your board.

POST /api/boards/{id}/cards
Create a new card
ParameterTypeDescription
column_id requiredstringColumn to add card to
title requiredstringCard title
description optionalstringCard description
Card Response
{
  "id": "card-uuid",
  "board_id": "board-uuid",
  "column_id": "column-uuid",
  "title": "Implement user authentication",
  "description": "Add JWT-based auth flow",
  "position": 0,
  "due_date": "2024-01-20T00:00:00Z",
  "assignees": ["user-uuid-1", "user-uuid-2"],
  "labels": ["label-uuid"],
  "created_by": "user-uuid",
  "created_at": "2024-01-15T10:30:00Z"
}
GET /api/cards/{id}
Get card details
PUT /api/cards/{id}
Update card
ParameterTypeDescription
titlestringCard title
descriptionstringCard description
due_datestringISO8601 date or null
assigneesarrayArray of user UUIDs
labelsarrayArray of label UUIDs
POST /api/cards/{id}/move
Move card to different column/position
ParameterTypeDescription
column_id requiredstringTarget column UUID
position requiredintegerPosition in column (0-indexed)
Move Card Example
curl -X POST http://localhost:8080/api/cards/CARD_ID/move \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"column_id": "done-column-uuid", "position": 0}'
DELETE /api/cards/{id}
Delete card

Example Workflow

Here's a typical workflow for setting up a project board:

1. Create a Board
curl -X POST http://localhost:8080/api/boards \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "Project Alpha", "description": "Q1 Development"}'
2. Create Columns
# Create "To Do" column
curl -X POST http://localhost:8080/api/boards/BOARD_ID/columns \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "To Do"}'

# Create "In Progress" column
curl -X POST http://localhost:8080/api/boards/BOARD_ID/columns \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "In Progress"}'

# Create "Done" column
curl -X POST http://localhost:8080/api/boards/BOARD_ID/columns \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "Done"}'
3. Create Labels
curl -X POST http://localhost:8080/api/boards/BOARD_ID/labels \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "Bug", "color": "#ef4444"}'
4. Create a Card
curl -X POST http://localhost:8080/api/boards/BOARD_ID/cards \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "column_id": "TODO_COLUMN_ID",
    "title": "Fix login bug",
    "description": "Users cannot login on Safari"
  }'
5. Update Card with Labels and Assignees
curl -X PUT http://localhost:8080/api/cards/CARD_ID \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "labels": ["BUG_LABEL_ID"],
    "assignees": ["USER_ID"],
    "due_date": "2024-01-20T00:00:00Z"
  }'

Tips

1. Use labels to categorize work types (Bug, Feature, Docs)
2. Set due dates for time-sensitive tasks
3. Assign team members to balance workload
4. Use comments for discussion instead of creating more cards
5. Reorder columns to match your team's workflow
